# pimono-demo 架构：极简 Agent 骨架

> 目标：用最少的代码（~700 行 / 8 主类）展示一个可运行的 ReAct Agent，无框架抽象。
> 适合作为学习起点；所有"花哨"特性（中间件 / 事件 / 权限 / 沙箱 / 团队）都到 agentscope-demo 才出现。

---

## 1. 模块定位

- **规模**：~700 行 / 8 主类 / 25 测试
- **依赖**：OkHttp（HTTP）、Jackson（JSON）、JLine3（REPL）、SLF4J（日志）
- **入口**：`com.demo.pimono.PiMonoDemoApplication`
- **目标读者**：想理解 "Agent 到底是什么" 的初学者

## 4. 五层通用 Agent 范式的映射

| 层 | pimono 实现 |
|---|---|
| ReAct 循环 | `AgentCore.chat()`：`while (round < MAX_TOOL_ROUNDS=50)` |
| LLM 客户端 | `LlmClient`（手写 OkHttp + Jackson） |
| 工具通道 | `McpClientManager`（5 mock builtin + MCP stdio 外接） |
| 上下文存储 | `ContextManager`（系统提示词 + FIFO 50 条滑窗 + ToolCallEntry） |
| tool_call_id 闭环 | `Context.ToolCallEntry`（id 关联 request 与 result） |

## 5. 数据流（单轮 chat）

```
用户输入
   │
   ▼
[ConsoleUI.readLine]
   │
   ▼
[AgentCore.chat]
   │
   ├─ contextManager.addUserMsg(userInput)
   │
   └─ while round < 50:
        ├─ messages = contextManager.buildMessages()  // 含 system + 历史
        ├─ tools = mcpClientManager.listTools()
        ├─ response = llmClient.chatCompletion(messages, tools)
        ├─ if !response.hasToolCalls: return response.text
        ├─ contextManager.addAssistantMsg(response)
        └─ for tc in response.toolCalls:
             ├─ result = mcpClientManager.executeTool(tc.name, tc.args)
             └─ contextManager.addToolResult(tc.id, result)
```

---

## 6. 关键不变量

1. **`tool_call_id` 闭环**：每条 toolCall 必须有对应的 toolResult，否则下一轮 LLM 调用会因协议错误失败。
2. **FIFO 50 条**：超出即丢最早，**不**做压缩（与 agentscope 不同）。
3. **同步阻塞**：整个 chat 单线程，无 CompletableFuture、无事件总线。
4. **MAX_TOOL_ROUNDS = 50**：与 `ARCHITECTURE.md` 写的 10 不符，以代码为准。

---

## 7. 设计取舍

| 选择 | 理由 |
|---|---|
| 手写 HTTP 客户端而非用 SDK | 教学透明，看得到 OpenAI 协议全貌 |
| FIFO 硬截断而非压缩 | 简单可预测；agentscope 演示压缩 |
| 无中间件 | 全部逻辑在 `AgentCore.chat` 一处可读 |
| 无权限系统 | 信任运行环境；agentscope 才引入三引擎 |
| mock 工具 | 不依赖外部服务即可演示 |
| 单上下文实例 | 教学场景足够；多会话需扩展 |

| 命令 | 行为 |

| `exit` / `quit` | 退出 |
| `history` | 打印上下文历史 |
| `status` | Agent 状态 |
| `clear` | 清空上下文 |
| `tools` | 列出可用工具 |
| `help` | 命令列表 |
| 其他文本 | 进入 `AgentCore.chat(...)` |
