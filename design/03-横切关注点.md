# 03 横切关注点：跨模块约定

> 不属于单一组件、但贯穿整个代码库的设计约定。

---

## 1. 日志（SLF4J + Logback）

**约定**：
- 一律使用 `org.slf4j.Logger`，禁止 `System.out.println` 进入业务代码。
- 用户可见输出走 `ui/ConsoleUI`（JLine3 ANSI 着色），与日志分离。
- 中文优先：日志消息以简体中文为主（与代码注释、UI 文案一致）。

**级别约定**：
| 级别 | 用途 |
|---|---|
| `ERROR` | 异常、工具执行失败、回复中断 |
| `WARN` | 权限拒绝、预算超限、降级路径 |
| `INFO` | 关键节点（Agent 创建、工具调用、团队生命周期） |
| `DEBUG` | 循环迭代、中间件触发、详细参数 |
| `TRACE` | 极少使用，仅调试深度流程 |

**示例**：
```java
log.info("智能体 [{}] 收到用户输入: {}", name, userInput);
log.warn("工具 [{}] 被权限引擎拒绝", toolCall.getName());
```

## 3. 中文优先

**适用范围**：
- 系统提示词（System Prompt）
- 用户可见的 UI 文案、命令提示
- 日志消息（INFO 及以上）
- 关键类的 JavaDoc 第一行

**例外**：
- 类名、方法名、变量名一律英文（Java 命名规范）
- 日志的 DEBUG/TRACE 细节可英文（性能考虑）
- 配置键（env var、JSON key）英文

**示例对照**：
| 类型 | 中文 | 英文 |
|---|---|---|
| 日志 | `智能体已创建` | — |
| 类名 | — | `Agent` |
| env var | — | `REPLY_BUDGET` |
| 异常消息 | `文件读取被拒绝` | — |
| JSON 工具 schema key | — | `"path"` |

## 4. 异常处理策略

### 4.1 检查 vs 非检查
- 业务可恢复异常 → 自定义检查异常（少用）
- 系统级/不可恢复 → `RuntimeException` 子类
- 工具执行异常 → 捕获后封装为 `ToolResult` 错误输出，**不**向外抛

### 4.2 关键自定义异常

| 异常 | 抛出点 | 处理 |
|---|---|---|
| `MCPClient.MCPInitException` | `MCPClient.initialize` 失败 | 应用启动终止 |
| `FilePermissionDeniedException` | FilePermissionManager 检查失败 | 工具捕获，转 `ToolResult` 错误 |
| `BudgetExceededException` | 历史：`ReplyBudgetControlMiddleware` | 2026-06 起不再抛出 |
| `RuntimeException("工具执行错误: ...")` | MCPClient.executeTool 内部 | 包装原始异常 |

### 4.3 循环内异常
`Agent.reply()` 的 `try/catch` 范围覆盖整个 ReAct 循环：
- 任何异常 → `progressTracker.onError` + `eventStream.emit(Error)` + 返回兜底消息
- 不再 rethrow，保证 REPL 不崩

## 5. 工具调用协议约定

### 5.1 统一接口
所有工具（builtin + MCP 外接）必须遵循：
```
{ name, description, inputSchema(JSON Schema) }
```
返回通过 `tool_call_id` 关联结果，**禁止**：
- 丢弃 `tool_call_id`
- 乱序匹配 request 与 result
- 使用独立的 "function calling" 旁路

### 5.2 工具注册方式
- pimono：`McpClientManager.registerBuiltinTools()` 硬编码
- agentscope：`MCPClient.registerCustomTool` / `registerFileTools` / `registerCodeExecutionTools`；外部通过 MCP stdio 配置 `MCP_SERVERS`

### 5.3 工具结果截断（agentscope 专有）
| 工具 | 上限（字符） |
|---|---|
| `agent_message` | 20000 |
| `agent_create` | 500 |
| `agent_list` | 5000 |
| 其他 | 5000 |

## 6. 线程模型

| 场景 | 模型 |
|---|---|
| pimono REPL | 单线程 |
| agentscope REPL（单 Agent） | 单线程 |
| agentscope `sendMessageToWorkersParallel` | `CompletableFuture.supplyAsync`，默认 ForkJoinPool |
| agentscope 事件流 | `CopyOnWriteArrayList`，线程安全 |
| agentscope MCP 连接 | 单连接非并发，多服务器聚合在 `ConcurrentHashMap` |

> 注意：当前未对并行工作者共享 Leader 上下文做加锁保护；多工作者同时操作同一文件可能产生竞态。

## 7. 依赖注入与装配

**无 Spring**，全部手写构造器装配：
- `main()` 是唯一装配点
- 依赖按"五层"顺序自底向上构造：凭证 → MCP → 工作空间 → 执行 → 权限 → Agent → 团队
- 修改装配时参考 `AgentScopeDemoApplication.main()` 的 9 步顺序

| 命名空间 | 风格 | 示例 |
|---|---|---|
| 类名 | UpperCamelCase | `AgentCore`、`ReplyBudgetControlMiddleware` |
