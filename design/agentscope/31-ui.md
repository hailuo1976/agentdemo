# 31 UI 模块（ui）

> 包：`com.demo.agentscope.ui`
> 主类：`ConsoleUI` + `AgentProgressTracker` + `TeamProgressTracker` + `VerbosityLevel`

---

## 1. 职责

终端交互层，提供：
- REPL 输入读取（JLine3）
- ANSI 彩色输出（横幅、消息框、状态卡）
- 进度跟踪（单 Agent + 团队）
- 详细程度分级（4 档 verbosity）

**不做**：业务逻辑、状态管理、网络。所有方法都是静态或纯展示。

---

## 2. ConsoleUI（静态工具类）

### 2.1 终端初始化

```java
static final LineReader STDIN_READER;
static {
    Terminal terminal = TerminalBuilder.builder().system(true).build();
    STDIN_READER = LineReaderBuilder.builder().terminal(terminal).build();
}
```

**为什么用 JLine3 而非 Scanner / BufferedReader**：
macOS 终端的 cooked mode 对 UTF-8 多字节字符（中文）退格回显有 bug——按一次退格只删 1 byte，导致屏幕显示和实际缓冲不一致。JLine3 自己接管 raw mode 并按字符（码点）处理退格，彻底解决。

### 2.2 ANSI 颜色常量

```
RESET / GREEN / BLUE / YELLOW / RED / CYAN / MAGENTA
BOLD / DIM
```

输出约定（与 pimono `ui/ConsoleUI` 一致）：
- 用户提示符 `You ▶` — 蓝
- Agent 输出框 `Agent ◀` — 绿
- 工具调用 — 青
- 思考 — DIM
- 错误 — 红
- 警告 — 黄
- 团队/领导者 — 品红

### 2.3 公开方法分类

| 类别 | 方法 |
|---|---|
| 横幅/提示 | `printBanner()` / `promptUser()` |
| Agent 响应 | `printAgentResponse(Msg)` |
| 事件流 | `printEventStream(EventStream)` |
| 历史记录 | `printHistory(List<Msg>)` |
| 状态 | `printAgentStatus(Map)` / `printTeamStatus(Map)` |
| 通用输出 | `printInfo` / `printSuccess` / `printError` / `printWarning` / `printSeparator` |
| 权限 | `printPermissionAsk(toolName, reason)` |
| 帮助 | `printHelp()` — 列出所有 REPL 命令 |

### 2.4 printAgentResponse 渲染规则

按 `ContentBlock` 类型分行渲染：
```
TextBlock      → 逐行原样输出
ThinkingBlock  → DIM 样式 "[思考] ..."
ToolCallBlock  → CYAN "🔧 调用工具: name" + "   参数: ..."
ToolResultBlock→ "✅ 结果" 或 "❌ 错误"（>120 字符截断）
```
末尾打印 token 用量（DIM）：`[tokens: prompt=X, completion=Y, total=Z]`。

### 2.5 printEventStream 渲染规则

每条事件一行：`<emoji> [HH:mm:ss] <summary>`。emoji 由 `eventIcon(EventType)` 映射（20 种），summary 由 `eventSummary(event)` 按 type 取关键 data 字段生成。

## 3. VerbosityLevel（4 档详细程度）

| 级别 | 显示内容 |
|---|---|
| `MINIMAL` | 仅最终结果 |
| `STANDARD`（默认） | 规划过程 + 关键里程碑 |
| `VERBOSE` | + 工具调用、执行细节 |
| `DEBUG` | + token 用量、内部状态、智能体间通信、时间线 |

### 3.1 来源优先级

```
System.getProperty("verbosity.level")    ← REPL 运行时覆盖（verbosity 命令）
  > 环境变量 VERBOSITY
  > 默认 STANDARD
```

`VerbosityLevel.fromEnv()` 解析失败时回落到 STANDARD。

### 3.2 `next()`

循环切换：`MINIMAL → STANDARD → VERBOSE → DEBUG → MINIMAL`。

REPL `verbosity` 命令通过 `next()` 实现"按一下切下一档"。

## 4. AgentProgressTracker（单 Agent 进度）

由 `Agent` 每次_reply_时新建（见 `27-agent-team.md` § 1.2）。

### 4.1 字段
```
String agentName;
VerbosityLevel verbosity;
int currentIteration;
long startTimeMs;
```

### 4.2 钩子

| 钩子 | 输出（按 verbosity） |
|---|---|
| `onReplyStart(userInput)` | STANDARD+: "🤖 <name> 思考中..."；VERBOSE+: 输入预览 |
| `onModelCallStart()` | STANDARD+: "📡 迭代 N: 模型推理中..." |
| `onModelCallComplete(prompt, completion)` | DEBUG: tokens 行 |
| `onToolCall(toolName)` | STANDARD+: "🔧 调用工具: ..." |
| `onToolCallComplete(toolName, success, ms)` | DEBUG: ✓/✗ + 耗时 |
| `onReplyComplete(ms)` | STANDARD+: "完成: N 次迭代, Xms" |
| `onError(error)` | 所有级别：红色 ❌ |

### 4.3 verbosity 守门模式

每个钩子开头先判 `verbosity`，MINIMAL 直接 return。DEBUG 钩子单独判 `== DEBUG`（不传播给更低级别）。

## 5. TeamProgressTracker（团队进度）

由 `AgentTeam` 持有，跨整个团队生命周期。

### 5.1 字段（线程安全）

```java
String teamId;
String leaderName;
Map<String, WorkerStatus> workers;       // ConcurrentHashMap
List<TimelineEntry> timeline;             // Collections.synchronizedList
Map<String, Set<String>> dependencies;   // ConcurrentHashMap
VerbosityLevel verbosity;
```

团队模式下多 worker 并发执行（`sendMessageToWorkersParallel`），所有写操作必须线程安全。

### 5.2 钩子

| 钩子 | 时机 |
|---|---|
| `onLeaderPlanningStart(goal)` | Leader 开始规划 |
| `onLeaderCreateWorker(name, role)` | `agent_create` 工具执行 |
| `onLeaderAssignTask(worker, task)` | `agent_message` 工具执行 |
| `onLeaderPlanningComplete(workerCount)` | Leader 规划结束 |
| `onWorkerStart(name, task)` | worker 开始执行 |
| `onWorkerToolCall(name, tool, iter)` | worker 内部工具调用（VERBOSE+） |
| `onWorkerComplete(name, success, ms)` | worker 任务结束 |
| `onAgentCommunication(from, to, msg)` | 智能体间通信（DEBUG） |

每个钩子都：
1. 更新 `WorkerStatus`
2. 追加 `TimelineEntry`
3. 按 verbosity 决定是否 `System.out.println`

### 5.3 WorkerStatus（嵌套类）

```java
class WorkerStatus {
    String name, role;
    TaskStatus status;            // PENDING / IN_PROGRESS / COMPLETED / FAILED
    String currentTask;
    LocalDateTime taskStartTime, taskEndTime;
    long durationMs;
    int toolCallCount;
    int currentIteration;
}
```

### 5.4 TaskStatus（嵌套枚举）

```
PENDING ⏳ → IN_PROGRESS 🔄 → COMPLETED ✅ / FAILED ❌
```

`printTeamOverview` 渲染时按状态映射 emoji。

### 5.5 TimelineEntry（嵌套类）

```java
class TimelineEntry {
    LocalDateTime time;
    String agent;
    String eventType;     // PLANNING / CREATE_WORKER / ASSIGN_TASK / TASK_START / TASK_COMPLETE ...
    String event;         // 人类可读描述
}
```

仅 `DEBUG` 模式下打印时间线（最近 10 条）。

## 6. 与 Agent / AgentTeam 的接线

### 6.1 单 Agent

```
Agent.reply(userInput):
  progressTracker = new AgentProgressTracker(name, verbosity)   ← 每次 reply 重建
  progressTracker.onReplyStart(userInput)
  for iter:
    progressTracker.onModelCallStart()
    response = chatModel.chat(...)
    progressTracker.onModelCallComplete(prompt, completion)
    if hasToolCalls:
      for tc: progressTracker.onToolCall(tc.name); ... progressTracker.onToolCallComplete(...)
  progressTracker.onReplyComplete(durationMs)
```

### 6.2 AgentTeam

```
AgentTeam.sendMessageToWorker(worker, msg):
  progressTracker.onLeaderAssignTask(worker, msg)
  progressTracker.onWorkerStart(worker, msg)
  worker.reply(msg)    ← worker 内部用自己的 AgentProgressTracker
  progressTracker.onWorkerComplete(worker, success, durationMs)
```

两层 tracker 协同：外层团队进度（Leader 视角）+ 内层单 Agent 进度（worker 视角）。

## 8. 与 pimono ConsoleUI 的对比

| 维度 | pimono | agentscope |
|---|---|---|
| 实现方式 | 静态工具类 | 静态工具类 + 进度跟踪器 |
| 输入读取 | JLine3 | JLine3（同样为退格 bug 选） |
| 进度跟踪 | 无 | AgentProgressTracker + TeamProgressTracker |
| 详细程度 | 无分级 | 4 档 VerbosityLevel |
| 事件流展示 | 无 | printEventStream（20 种 emoji） |
| 团队状态 | 无 | printTeamStatus / printTeamOverview |
| 权限展示 | 无 | printPermissionAsk |

## 9. 已知限制

- **输出耦合 System.out**：所有方法直接 `System.out.println`，不可重定向（生产化应引入 `PrintStream` 字段）
- **无 TTY 检测**：管道/重定向场景下 ANSI 转义会污染输出
- **emoji 宽度**：在某些终端（Windows cmd.exe）emoji 宽度计算不准，导致边框对不齐
- **进度跟踪器非线程封闭**：parallel 场景下多 worker 共享同一 `TeamProgressTracker`，靠并发集合保证安全，但输出顺序非确定

## 10. 测试

UI 类通常不写单元测试（输出难断言）。建议覆盖：
- `VerbosityLevelTest`：`fromEnv` 优先级、`next()` 循环、非法值兜底
- `AgentProgressTrackerTest`：verbosity 守门逻辑（捕获 System.out 验证）
- `TeamProgressTrackerTest`：并发 emit、状态枚举转换、时间线追加
